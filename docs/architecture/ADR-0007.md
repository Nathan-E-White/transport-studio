# ADR 0007: Staged Relativistic Multiphysics Kernel Promotion

## Status

Proposed for staged implementation.

## Context

Transport Studio has a V1 physics roadmap and an early `spacetime-physics` substrate. The next major physics direction is 3D general-relativistic radiation hydrodynamics, but that phrase is too broad to implement honestly in one slice.

The project needs an architecture decision that records the formulation stack, preserves product claim boundaries, and names the first implementation seam before code changes begin.

## Decision

Adopt Relativistic Multiphysics as the umbrella domain and make Dynamical Spacetime Coupling the first implementable slice.

The first implementation remains crate-kernel work in `spacetime-physics`. Product-facing runnable solver promotion remains gated until backend capability metadata, validation evidence, reporting, and UI copy all move together.

The staged formulation choices are:

- spacetime evolution: BSSN
- gauge language: 1+log slicing with Gamma-driver shift
- hydrodynamics: Valencia GRHD
- radiation: gray M1 moment evolution
- AMR: AMR-aware single-block start
- Monte Carlo packets: bridge for deposition/probe workflows, not primary radiation evolution

## TDD Seams

The first implementation should use these seams:

- kernel seam: one coupled deterministic step with explicit inputs, outputs, and diagnostics
- primitive recovery seam: conservative-to-primitive inversion and failure reporting
- grid metric adapter seam: sampling geometry state for matter/radiation updates
- packet deposition seam: packet-carried source accumulation into continuum fields
- BSSN/constraint diagnostics seam: constraint checks and algebraic projections around geometry updates
- single-block AMR adapter seam: block metadata and refinement diagnostics without multi-block regridding

## Consequences

- The project gets shared language before implementation.
- Later code can test kernel behavior without promoting an unsupported product solver.
- BSSN, Valencia GRHD, gray M1, AMR, and packet deposition stay distinct instead of collapsing into one vague "GR rad-hydro" label.
- The Monte Carlo heritage remains useful without forcing sampled packets to masquerade as the primary continuum radiation model.
- Review can focus on whether each seam has evidence instead of arguing about the whole future solver at once.

## Non-Goals

- Do not expose a new runnable Relativistic Multiphysics solver in this decision.
- Do not claim production GR radiation-hydrodynamics capability.
- Do not claim strong-field production numerical-relativity capability.
- Do not promote primary Monte Carlo radiation evolution, full Berger-Oliger AMR, curvilinear chart support, GRMHD, TOV/static-star validation, Bondi/Michel validation, AMR convergence, strong-field constraint preservation, GRMHD tests, or tabulated EOS as part of the first slice.
- Do not implement multi-block AMR in the first slice.
- Do not replace existing Monte Carlo transport paths.
- Do not merge gray M1 radiation fields, Monte Carlo packet tracks, and tallies into one undifferentiated result stream.
- Do not expand criticality, depletion, or weapon-effects modeling under this roadmap.

## Follow-Up Work

- Add kernel tests at the named seams.
- Implement the smallest deterministic Dynamical Spacetime Coupling kernel that can satisfy those tests.
- Add capability metadata only when diagnostics and validation language are ready.
- Add report and UI language only when the solver can distinguish solved, placeholder, benchmark, and unsupported facets.
